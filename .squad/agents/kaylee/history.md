# Project Context

- **Owner:** Dovi Joel
- **Project:** Sefaria Book Creator Web App — a full-stack web application that lets users create beautifully typeset PDF books from Jewish texts on the Sefaria platform. Based on the Python/LaTeX pipeline in `nkasimer/books-from-sefaria` (forked to `dovijoel/books-from-sefaria`).
- **Stack:** Next.js 14 (App Router, TypeScript, Tailwind CSS, shadcn/ui) · FastAPI (Python 3.11) · Celery + Redis (async jobs) · PostgreSQL · LaTeX/texlive (PDF generation) · Docker Compose
- **Created:** 2025-06-16

## Learnings

<!-- Append new learnings below. Each entry is something lasting about the project. -->

- The book settings have a LOT of configurable fields — typography, margins, paper size, fonts, layout, cover colors, etc. The UI needs to be well-organized (tabs or collapsible sections) or it'll be overwhelming.
- Hebrew text support is critical — RTL layout, Hebrew font selection, and Hebrew page number support.
- PDF generation is async (30s–3min) — the UI needs a proper job status polling mechanism with progress indication.
- The frontend communicates with the FastAPI backend via REST API (OpenAPI spec generated by FastAPI).
